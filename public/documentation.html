Introduction

Room Correction Helper (RCH) est un outil de calibration pour installations home cinema Denon et Marantz basees sur Audyssey. Il travaille avec REW pour importer des mesures, aligner les enceintes et les caissons, generer les filtres, verifier les previews et produire un fichier OCA exploitable par votre AVR.

RCH n'est pas un bouton magique. Le principe de l'application est au contraire de vous proposer un parcours guide, mais transparent, pour que vous puissiez verifier chaque etape avant l'export final.

Cette documentation decrit le fonctionnement actuel de l'application et suit les commandes visibles dans l'interface.

Cette aide est integree directement dans l'application. Les indications ci-dessous supposent donc que vous etes deja dans RCH et que vous pouvez revenir a l'onglet Application a tout moment.

Avant de commencer

Avant d'utiliser RCH, verifiez les points suivants :

  • Vous disposez d'un AVR Denon ou Marantz compatible Audyssey.
  • Vous avez installe une version beta recente de REW.
  • Le serveur API de REW est active sur le port 4735.
  • Toutes les operations de calcul necessitent une connexion active a REW. Sans connexion, RCH peut encore analyser certains fichiers et proposer des exports, mais il ne pourra pas lancer les traitements.
  • Vous avez un point d'entree compatible avec le flux actuel : ADY pour le mode guide, ou AVR + mesures REW pour le mode avance.
  • Pour le mode avance, vous savez deja mesurer correctement dans REW avec un micro de mesure ou une methode equivalente.

Si vous prevoyez de generer puis de charger un fichier OCA dans l'AVR, odd.wtf reste l'outil le plus naturel pour ce flux de travail.

Les binaires odd.wtf par systeme, les sweeps lossless et les fichiers Atmos restent centralises dans l'onglet Resources. Cette page se concentre donc sur le moment ou il faut les utiliser, pas sur une liste complete de fichiers a telecharger.

Conseil pratique : gardez un dossier de travail propre avec vos fichiers ADY, MQX, AVR, MDAT, OCA et courbes cibles. Cela evite beaucoup d'erreurs de protocole et de manipulation de fichiers.

Si vous utilisez une courbe cible personnalisee, preparez-la a l'avance. Vous en aurez besoin dans REW pour Align SPL puis pour valider le resultat final.

Choisir votre point de depart

1. Mode simple avec fichier ADY

C'est le point d'entree le plus simple. Vous mesurez avec odd.wtf ou avec l'application Audyssey MultEQ Editor, puis vous importez le fichier ADY dans RCH. Quand RCH est connecte a REW, les mesures sont chargees automatiquement dans REW.

Avec odd.wtf, le flux le plus simple reste la mesure integree qui produit directement un fichier ADY. Si vous preferez mesurer vous-meme dans REW avec votre propre micro, gardez odd.wtf uniquement pour generer le fichier AVR du mode avance.

Si REW n'est pas connecte, RCH peut quand meme extraire les mesures du fichier, mais l'import automatique dans REW ne se fera pas.

2. Cas particulier du fichier MQX

RCH sait aussi traiter les fichiers MQX, mais ce flux ne constitue pas aujourd'hui le parcours standard le plus robuste dans la page Application.

En pratique, retenez surtout ADY comme point d'entree simple, et AVR + REW comme point d'entree avance. Si vous travaillez avec MQX, considerez-le comme un cas avance a valider avec prudence selon votre projet et votre contexte AVR.

Pour utiliser ce flux, chargez d'abord les donnees AVR. RCH en a besoin pour reconstruire correctement la correspondance des canaux.

3. Mode avance avec REW

Si vous mesurez vous-meme dans REW, vous devez generer un fichier AVR avec odd.wtf, puis charger cet AVR dans RCH. Les noms de vos mesures REW doivent suivre les noms Audyssey exposes par le fichier AVR, sinon elles ne seront pas associees aux bons canaux dans le tableau de parametres.

La sequence la plus fiable est la suivante : lancer REW vide, demarrer son API, connecter RCH, puis seulement ouvrir le fichier MDAT dans REW. Avant de calculer, verifiez aussi que vos mesures n'ont pas conserve d'EQ, de delais ou d'inversions parasites.

Preparation recommandee pour des mesures manuelles dans REW

Avant la campagne de mesures, repartez d'un AVR dans un etat simple et previsible. Le minimum utile est le suivant :

  • Supprimez ou neutralisez une correction precedente avant de mesurer.
  • Reglez le volume autour de 62 ou -18 dB, avec environ 75 dB sur Front Right.
  • Coupez les traitements inutiles pendant la mesure, notamment l'ECO mode et les traitements de loudness.
  • Si vous mesurez les subs via le chemin LFE, montez provisoirement LPF for LFE au maximum, ou prevoyez d'utiliser Revert plus tard dans RCH.
  • Pour un systeme Atmos, utilisez les sweeps lossless correspondants depuis Resources, avec Front Left comme reference d'acoustic timing.

Cote REW, gardez une configuration stable : Java EXCL si votre interface le permet, echantillonnage a 48 kHz, plage 0-24000 Hz et sweeps a -12 dBFS pour rester coherents avec les canaux Atmos.

Generer un fichier AVR avec odd.wtf

En mode avance, le fichier AVR sert de reference de nommage pour faire correspondre les mesures REW aux canaux Audyssey.

  1. Telechargez le binaire adapte a votre OS depuis l'onglet Resources et placez-le dans un dossier stable.
  2. Sur macOS ou Linux, rendez-le executable si necessaire, puis lancez d'abord une commande d'aide pour verifier qu'il demarre correctement.
  3. Utilisez ensuite la commande de recherche reseau pour detecter l'AVR, puis la commande de generation pour produire le fichier AVR a importer dans RCH.
  4. Si la detection automatique echoue, relancez la generation en precisant directement l'adresse IP de l'AVR.
odd search
odd gen
odd gen -i 192.168.x.x

Le nom exact du binaire depend de votre systeme. Adaptez simplement la commande a odd.wtf, odd.mac ou odd.linux selon votre cas. L'application Denon ou Marantz sur smartphone permet en general de retrouver rapidement l'adresse IP de l'AVR.

Parcours recommande

Etape 1. Connecter REW et charger le projet

  1. Lancez REW et demarrez son serveur API.
  2. Revenez a l'onglet Application.
  3. Cliquez sur Connect dans Main Commands.
  4. Glissez-deposez votre fichier ADY ou AVR.
  5. Si vous etes en mode avance, ouvrez ensuite votre jeu de mesures dans REW pour que RCH puisse les retrouver.

Une fois le projet charge, ouvrez les details AVR et verifiez que les canaux detectes correspondent bien a votre configuration.

En mode avance, evitez de charger le MDAT avant la connexion. Le tableau de RCH se remplit de facon plus fiable quand REW est deja lance, API active, puis fichier charge apres avoir clique sur Connect.

Etape 2. Reinitialiser proprement l'etat de calcul

Utilisez Reset Measurements si vous voulez reinitialiser toutes les mesures avec les parametres par defaut avant de relancer les calculs. Utilisez Reset Application si vous voulez repartir d'un projet vide apres une mauvaise manipulation ou un changement complet de projet.

En pratique, Reset Measurements remet notamment a zero le lissage, les fenetres IR, les parametres de cible, la room curve, l'equalizer generique, les delais et les inversions. C'est le bon reflexe avant de refaire un passage complet.

Etape 3. Regler les options de calcul pour les enceintes

Dans Speaker Tools, vous pouvez regler le lissage, le fenetrage, les limites de boost et la plage de frequence. Le lissage et le fenetrage pilotent la finesse de l'egalisation — un lissage leger produit des corrections detaillees avec beaucoup de filtres, un lissage fort des corrections subtiles avec peu de filtres — ainsi que l'affichage des courbes. Les calculs d'alignement des caissons (Align Sub, somme predite, plafond theorique) mesurent quant a eux toujours la reponse brute, independamment de ces reglages : vos choix d'affichage ne peuvent pas degrader la precision de l'alignement.

Le fenetrage Optimized MTW isole le son direct et les resonances : ideal pour choisir quoi corriger. Il abaisse toutefois la courbe de 1 a 2 dB au-dessus de 300 Hz ; RCH mesure et compense desormais automatiquement ce decalage au moment du calcul, bande par bande — le dosage des filtres reste ainsi juste, et la pente de votre courbe cible est bien transmise au resultat (l'offset applique figure dans les logs). Si vous preferez une courbe detaillee dans les basses et lisse dans les aigus sans modification de niveau, le lissage variable est l'alternative recommandee.

Dans le meme esprit, RCH corrige finement le grave et lisse fortement l'aigu : dans l'aigu, le son direct est deja bien sous la cible, et ne pas le sur-corriger est la pratique recommandee. La plage de frequence que vous reglez fait foi — RCH n'impose pas de zone. Gardez en tete que c'est le cumul mesure corrigee + chaine reelle qui doit suivre la cible : evitez la sur-correction.

Etape 4. Faire les moyennes

Si vous avez des mesures sur plusieurs positions, commencez par les moyenner avec Averages. Trois strategies sont disponibles : supprimer toutes les originales, conserver uniquement la position 1, ou tout garder.

Conserver la position 1 en plus des moyennes est utile si vous voulez ensuite comparer le resultat au point d'ecoute principal sans perdre les courbes moyennees.

RCH propose trois methodes de moyenne : Vector average, RMS + phase avg. et dB + phase avg. Si vous n'avez pas encore de preference, choisissez une methode et gardez la meme sur tout le projet pour que les comparaisons restent coherentes.

Avant chaque moyenne, RCH aligne desormais lui-meme les positions dans le temps (correlation d'enveloppes pour degrossir, puis correlation fine contrainte), a la place du Cross corr align de REW qui pouvait se decaler d'un cycle entier sur certaines mesures. Les decalages appliques sont indiques dans les logs. Cet alignement ameliore la coherence entre positions, en particulier pour le Vector average et les subwoofers.

Pour le calcul des filtres, preferez RMS + phase avg. : cette methode preserve le niveau a toutes les frequences. Le Vector average reste precieux pour l'alignement temporel et pour juger la coherence entre positions, mais il sous-estime le niveau de la mesure (de -1 a -2 dB avec 3 positions, jusqu'a -5 dB avec 8 positions) : des filtres calcules dessus seront sous-doses.

Etape 5. Aligner les enceintes dans le temps

Utilisez Time Align pour recalculer les distances et remettre les enceintes en phase temporelle. Cette etape doit etre faite avant les reglages fins de niveau et avant le travail sur les raccords enceintes-caisson.

Le point de reference n'est plus le pic de l'impulsion mais le temps d'arrivee de l'onde, estime par la phase en exces (la methode « Estimate IR delay » de REW) : le pic peut tomber sur une reflexion plus forte que le son direct et fausser la distance de plusieurs metres, la ou l'arrivee reste stable. Les distances obtenues sont donc plus proches des distances physiques ; en cas d'impulsion indisponible, RCH revient a l'ancien alignement sur le pic (signale dans les logs).

Une fois le Time align fait, les enceintes ne bougent plus : seuls les subs seront deplaces ensuite (Find Sub Alignment). Ne relancez pas d'alignement temporel des enceintes apres cette etape sans raison, car les etapes suivantes en dependent.

Cas des enceintes Atmos a reflexion (posees a plat sur les frontales, orientees vers le plafond) : le chemin utile est le rebond au plafond, pas le petit son direct qui fuit vers le micro. L'alignement et la distance suivent ce rebond ; verifiez la coherence de la distance sur ces canaux.

Etape 6. Charger la courbe cible puis faire Align SPL

La courbe cible doit etre chargee dans REW avant d'utiliser Align SPL. Cette etape ajuste les niveaux des enceintes par rapport a cette cible.

Important : faites Align SPL sur la bonne position. En pratique, cela signifie le plus souvent la position moyennee, pas une mesure brute prise isolenment.

Si RCH ne trouve pas votre cible, chargez-la dans REW via EQ / Target settings / House curve, puis verifiez que son niveau de reference reste bien a 75 dB.

Etape 7. Regler les frequences de coupure et generer les filtres

Dans le tableau de parametres, choisissez une frequence de coupure globale ou par groupe d'enceintes, puis utilisez les boutons Create sur chaque ligne ou All Filters pour tout generer d'un coup.

L'idee est de choisir des coupures realistes par rapport a la reponse des enceintes et aux capacites de raccord avec le sub. Une coupure trop basse peut rendre l'alignement plus difficile et produire un raccord mediocre.

Pour une paire symetrique comme FL et FR, gardez de preference la meme frequence de coupure tant qu'une mesure ne justifie pas un traitement different.

Les filtres sont calcules par RCH. Si vous preferez les generer ou les ajuster directement dans REW (EQ / Match target, ou edition manuelle), RCH detecte que les filtres ont change et utilise ceux presents dans REW pour toute la suite du process : previews, alignement du sub et exports.

Le calcul lui-meme est pilote par les reglages du panneau Settings. Les valeurs par defaut sont concues pour donner un bon resultat sans rien toucher : si vous voulez comprendre ce qui se passe quand vous cliquez sur Create, ou affiner le comportement, tout est detaille dans Le calcul des filtres (AutoEQ).

Trouver automatiquement le meilleur crossover

A cote de la liste de coupure, le bouton en forme de baguette lance une recherche automatique du meilleur crossover pour le groupe d'enceintes. RCH teste chaque frequence candidate, calcule le required shift de chaque enceinte du groupe (la meme mesure que le bouton de recalcul de la colonne Required shift), puis selectionne automatiquement dans la liste celle qui minimise la moyenne, en valeur absolue, des required shift des enceintes du groupe. Un bouton identique dans l'entete de la colonne fait la recherche pour tous les groupes d'un coup.

Comme le crossover est une propriete de groupe, le bouton par ligne n'apparait que sur la premiere enceinte de chaque groupe. Cliquer dessus traite tout le groupe.

Prerequis : les filtres doivent deja etre generes et la somme des subs doit exister (Sum subs / LFE predicted), sinon RCH affiche une alerte et ne fait rien. En pratique : generez une premiere fois les filtres avec une coupure de depart, faites la somme des subs, puis lancez la recherche du meilleur crossover et regenerez les filtres pour la valeur retenue.

Si le meilleur crossover demande d'inverser la polarite, RCH applique l'inversion tout de suite, et toujours de facon coherente sur toutes les enceintes du groupe : une frequence qui inverserait une seule enceinte d'une paire est ecartee. Le journal detaille, pour chaque frequence testee, le shift de chaque enceinte, la moyenne, l'inversion eventuelle (inv) et les frequences ecartees.

Changer de crossover rend l'aval a refaire (filtres, Find Sub Alignment, previews) : le journal le rappelle.

Etape 8. Aligner le sub avec une enceinte

Dans Tuning, selectionnez l'enceinte de reference puis utilisez Find Sub Alignment. L'enceinte choisie doit deja avoir son filtre calcule. En pratique, on commence le plus souvent par FL, FR ou la centrale.

Si le resultat est mauvais, essayez une autre enceinte, ou relevez la frequence de coupure avant de relancer la recherche d'alignement.

Find Sub Alignment deplace le bloc des subs d'un seul delai, dans une plage bornee (environ une demi-periode du crossover), et avance volontairement le sub d'un seizieme de periode. Un petit residuel non nul de required shift apres l'alignement est donc normal.

Quand l'alignement reussit, RCH met aussi a jour LPF for LFE avec une valeur coherente avec l'enceinte retenue, avec un minimum de 120 Hz. Verifiez quand meme ce reglage avant l'export final.

Etape 9. Verifier les previews

Utilisez les boutons Preview ligne par ligne, ou All previews, pour verifier les courbes prevues enceinte + sub avant export. C'est la meilleure facon de detecter un mauvais raccord, une inversion utile ou un crossover mal choisi.

Si necessaire, le bouton Auto Adjust Inversion peut aider a trouver une polarite plus favorable sur le raccord avec le sub.

Attention : si une enceinte demande vraiment une inversion de polarite pour bien se raccorder, cela peut trahir une erreur de cablage (fils + et - inverses). Verifiez-la physiquement plutot que de la masquer par le DSP.

Si les previews sont bons sur la moyenne mais moins convaincants sur la position d'ecoute principale, refaites la verification en conservant aussi la position 1 au moment du moyennage.

Note sur le raccord : le bass management de l'ampli est asymetrique — il applique un passe-bas LR24 au sub mais seulement un passe-haut BW12 a l'enceinte, en pariant que la decroissance naturelle de l'enceinte complete ce BW12 — or l'egalisation a justement aplani cette decroissance, ce qui ferait depasser la cible d'environ 2 dB autour du crossover. RCH corrige cela d'office : un second BW12 est integre aux filtres OCA de chaque enceinte avec bass management, si bien que le raccord reel devient symetrique LR24|LR24, en phase a toutes les frequences. Les previews et Find Best LFE Low-Pass simulent cette chaine complete : ce que montre la preview est ce qui sortira de l'ampli. Les mesures d'alignement (Check, Find Sub Alignment, Find Best Crossover) travaillent, elles, sur les courbes egalisees brutes — la paire LR24|LR24 etant en phase a toutes les frequences, elle n'invalide pas ces mesures. Point a surveiller dans les previews : sous le crossover, la ou l'enceinte decroit naturellement sans que l'egalisation ait pu compenser, le second BW12 accentue legerement les creux — c'est la zone ou les subs dominent, et un crossover bien choisi (Find Best Crossover) evite les cas defavorables.

Le calcul des filtres (AutoEQ)

Quand vous cliquez sur Create, RCH calcule lui-meme les filtres parametriques de la mesure : c'est le moteur AutoEQ. Cette section explique ce qu'il fait, ce que chaque reglage change, et comment lire son verdict. Vous n'avez rien a regler pour obtenir un bon resultat : les valeurs par defaut sont un compromis eprouve sur les mesures de reference. Lisez la suite quand vous voulez comprendre, ou affiner.

Ce que fait le moteur

Le principe est simple : RCH compare votre mesure a la courbe cible, et pose des filtres la ou l'ecart le justifie. Dans le detail, le calcul se deroule toujours dans cet ordre :

  1. Analyse de l'ecart. RCH decoupe la mesure sur la plage de frequence que vous avez choisie, ramene la cible sur la meme grille, puis identifie les zones qui s'ecartent de la cible. Les creux tres profonds sont mis de cote : ce sont des annulations acoustiques, les corriger ne sert a rien.
  2. Placement des filtres. RCH pose les filtres un par un. A chaque etape il recalcule ce qu'il reste a corriger, choisit la zone la plus utile, place un filtre dessus et optimise son gain et son Q. Il s'arrete des qu'aucune zone ne justifie plus un filtre, meme s'il reste des emplacements libres.
  3. Nettoyage et reoptimisation. Les filtres devenus inutiles sont retires, et ceux qui restent sont reoptimises ensemble. Cette boucle se repete jusqu'a ce qu'il n'y ait plus rien a enlever.
  4. Reduction des depassements. Si la correction fait passer la reponse trop au-dessus de la cible quelque part, RCH baisse le filtre de boost responsable.
  5. Options avancees. Selon les cases cochees, RCH peut rejouer le placement en testant plusieurs hypotheses, puis polir le resultat.

Un point important : a chaque etape, RCH ne garde le nouveau resultat que s'il est meilleur que le precedent. Une option activee ne peut donc pas degrader le resultat retenu par RCH ; au pire elle ne change rien, en echange du temps de calcul.

Ou se trouvent les reglages

Les reglages du moteur sont dans le panneau Settings, section AutoEQ. Trois autres reglages, situes dans Speaker Tools, pesent tout autant sur le resultat : les deux limites de boost et la plage de frequence. Ce sont eux qu'il faut regarder en premier.

Individual Max Boost et Overall Max Boost

Le premier limite ce qu'un filtre seul peut amplifier ; le second limite ce que tous les filtres cumules peuvent amplifier a une meme frequence. Amplifier coute de la puissance a l'ampli et de l'excursion aux haut-parleurs : ces deux limites sont la pour proteger votre materiel. Les valeurs par defaut, 6 dB et 3 dB, sont deja un bon compromis. Les deux champs acceptent 0 a 6 dB.

Frequency Range

La plage que vous reglez fait foi : RCH ne corrige rien en dehors et n'impose aucune zone interdite. C'est votre principal outil de controle. Par defaut la plage s'arrete a 16 kHz, volontairement : dans l'aigu, le son direct est deja sous la cible et une mesure en un point ne represente plus ce que vous entendez. Elargir jusqu'a 20 kHz est possible, mais rarement souhaitable.

Les curseurs, dans l'ordre de l'ecran

Filters (defaut 20)

Nombre maximal de filtres que le moteur peut utiliser. C'est un plafond, pas un objectif : RCH n'en pose que ce qui est justifie, et retire ceux qui n'apportent rien. Autoriser beaucoup de filtres permet de corriger des accidents etroits ou multiples, mais donne une correction plus complexe ; peu de filtres donnent une correction plus simple et plus robuste.

Max cut (dB) (defaut 25)

Attenuation maximale d'un filtre. Attenuer ne coute rien a l'ampli, contrairement a amplifier : c'est pourquoi cette limite est bien plus large que celles de boost. La correction par attenuation est la plus sure.

Flatness target (defaut 0.3)

Seuil a partir duquel un ecart a la cible est juge significatif, en dB. C'est le reglage qui decide de ce que le moteur considere comme un defaut a corriger. Plus la valeur est basse, plus RCH traite de petits ecarts, donc plus il pose de filtres ; plus elle est haute, plus il se concentre sur les gros defauts. C'est aussi ce seuil qui finit par arreter le placement : quand plus aucune zone ne le depasse, le placement s'arrete.

Optimization passes (defaut 20)

Nombre maximal de passes de la boucle nettoyage + reoptimisation d'ensemble (ce n'est pas un nombre de passes par filtre). La boucle s'arrete d'elle-meme des qu'une passe n'a plus rien a retirer, ce qui arrive presque toujours des la premiere : augmenter cette valeur ne change en general rien au resultat ni au temps de calcul.

Gain sign lock (defaut 0.5)

Seuil, en dB, a partir duquel le signe d'un filtre est verrouille : au-dela, un filtre qui attenue ne pourra plus devenir un filtre qui amplifie, et inversement. Cela evite qu'un filtre hesite entre les deux pendant l'optimisation. Valeur basse = verrouillage plus rapide, comportement plus stable ; valeur haute = plus de liberte laissee a l'optimiseur.

Notch exclusion (defaut 6)

Profondeur, en dB sous la cible, a partir de laquelle un creux est declare non corrigeable et exclu du calcul. C'est un reglage important : la plupart des creux profonds viennent d'annulations acoustiques entre le son direct et une reflexion, et les combler demanderait une energie enorme pour un resultat nul, voire dangereux pour le materiel. Valeur elevee = RCH tente de corriger davantage de creux ; valeur basse = il en ignore plus.

Min filter gain (defaut 0.4)

Gain minimal, en dB, pour qu'un filtre soit conserve et envoye a REW. En dessous, le filtre est juge decoratif et supprime. Cela evite d'encombrer l'egaliseur avec des filtres qui n'ont aucun effet audible.

Refinement iterations (defaut 100)

Nombre d'iterations de la passe finale de polissage. N'a d'effet que si la case Refinement est cochee.

No boost below (Hz) (defaut 50)

Aucun filtre place sous cette frequence ne pourra amplifier : la zone devient attenuation seule. C'est une protection materielle. Sous 50 Hz, un boost demande beaucoup d'excursion au haut-parleur et beaucoup de puissance a l'ampli, pour corriger le plus souvent une annulation que l'on ne corrigera pas. Mettre 0 desactive la protection.

Overshoot penalty weight (defaut 0.3)

Un overshoot est un depassement de la courbe cible : la correction fait passer la reponse au-dessus de la cible, ce qui s'entend souvent plus qu'un creux. Ce curseur decourage l'optimiseur de produire de tels depassements, au-dela de 1 dB. Plus le poids est eleve, plus il les evite, quitte a etre moins precis ailleurs.

Overshoot reduction threshold (dB) (defaut 1.5)

Le meme sujet, traite autrement : une fois les filtres calcules, tout depassement de la cible superieur a ce seuil est corrige en baissant le filtre de boost responsable. Le curseur precedent previent pendant le calcul, celui-ci repare apres. C'est aussi le seuil au-dela duquel le rapport affiche un avertissement.

Max Q below 200 Hz et Max Q in high band (defaut 0 = desactive)

Ces deux curseurs plafonnent le facteur Q, c'est-a-dire l'etroitesse des filtres. Un filtre etroit est un resonateur : plus son Q est eleve, plus il met de temps a s'eteindre, ce qui peut s'entendre comme une traine (ringing). Au-dela de Q = 8, un filtre correctif devient audible en tant que tel : c'est pourquoi la plage s'arrete la. Laisses a 0, ces plafonds sont inactifs et seules les regles internes s'appliquent.

Les deux bandes ne posent pas le meme probleme. Dans le grave, un filtre etroit qui attenue un mode de piece est utile : il raccourcit la resonance de la piece au lieu d'en creer une. Plafonner trop bas sous 200 Hz vous prive de ce traitement. Dans l'aigu en revanche, un filtre etroit corrige surtout une interference locale, qui change des que le micro bouge : la, plafonner est generalement une bonne idee. Si vous ne devez en regler qu'un, reglez celui de la bande haute, vers 2 ou 3.

High band starts at (Hz) (defaut 3000)

Frequence a partir de laquelle le plafond Max Q in high band s'applique. Le defaut de 3 kHz correspond a la zone ou les mesures en un point deviennent peu representatives. Baissez-le si vous voulez etendre la prudence au medium.

Les cases a cocher

Beat-REW optimization (decoche par defaut)

Active un mode d'optimisation plus ambitieux, oriente qualite finale plutot que stricte parite avec REW. Plus lent, potentiellement meilleur.

A savoir : cette case commande aussi les deux suivantes. Reduce & repair et Critical band refinement sont indentees sous elle et restent grisees (sans effet) tant que Beat-REW optimization est decochee. Si vous voulez en profiter, c'est cette case qu'il faut cocher.

Reduce & repair (cochee, necessite Beat-REW)

Apres une premiere solution, RCH essaie d'enlever les filtres les moins utiles ou les plus risques, puis de reoptimiser ce qui reste. Le but est une correction plus propre a qualite egale : moins de filtres, meme resultat.

Critical band refinement (cochee, necessite Beat-REW)

Reoptimise separement trois bandes — le grave modal, le medium critique et l'aigu — au lieu de ne viser qu'un score global. L'idee est de mieux equilibrer la qualite la ou l'oreille est la plus sensible.

Candidate placement (cochee par defaut)

Au lieu de placer chaque filtre sur la premiere zone venue, RCH teste plusieurs zones candidates et garde la plus prometteuse. Concretement, RCH rejoue tout le placement de cette maniere et ne remplace le premier resultat que si le second est meilleur. Cela double le temps de calcul, mais ne peut pas degrader le resultat : c'est pourquoi c'est active par defaut.

Refinement (decoche par defaut)

Passe finale de polissage sur la grille complete, alors que le reste du calcul travaille sur une grille allegee. Sur les exemples de reference, cette option ameliore le RMS partout, mais degrade les depassements dans les pieces a creux marques. Elle est donc laissee decochee et a activer au cas par cas, en comparant.

Vary Q above 200 Hz (decoche par defaut)

Fait varier progressivement le plafond de Q avec la frequence au-dessus de 200 Hz, au lieu d'un plafond fixe. C'est la preference correspondante de REW, laissee sur la meme valeur par defaut que dans REW.

Narrow filters below 200 Hz (cochee par defaut)

Autorise les filtres etroits dans le grave, la ou les resonances de piece sont etroites et marquees. Decocher elargit les corrections du grave, ce qui les rend plus douces mais moins efficaces sur les modes.

Allow boosts (cochee par defaut)

Autorise RCH a creer des filtres qui amplifient pour remonter les zones sous la cible. Decocher travaille en attenuation seule, l'approche la plus sure.

A savoir : decocher cette case grise les deux limites de boost (Individual Max Boost et Overall Max Boost) et le calcul se fait comme si elles valaient 0. Vos valeurs ne sont pas perdues : elles redeviennent actives des que vous recochez la case.

Reset defaults

Ce bouton n'est pas un reglage acoustique : il remet tous les parametres AutoEQ a leurs valeurs par defaut. C'est le bon reflexe si vous vous etes perdu dans les reglages.

Lire le resultat

A la fin de chaque calcul, RCH publie un verdict dans les logs, sous la forme [createPhaseMatchFilter] rapport: PASS, suivi le cas echeant des filtres signales. Trois verdicts sont possibles :

  • PASS : rien a signaler, la correction est saine.
  • WARN : le resultat est utilisable mais merite un coup d'oeil. Le plus souvent : un filtre au Q eleve (au-dela de 8 dans le grave, de 10 ailleurs), un boost au-dessus de 3 kHz, un depassement de cible residuel, une amelioration inferieure a 10 %, ou un boost cumule au-dessus de votre limite globale.
  • FAIL : a ne pas utiliser tel quel. Soit un filtre depasse un plafond de securite (Q superieur a 10 dans le grave, a 12 ailleurs), soit la correction degrade la reponse au lieu de l'ameliorer.

Un FAIL n'est presque jamais un probleme de reglage : c'est le signe que la mesure demande quelque chose d'impossible. Les bons reflexes sont d'aller voir du cote de la mesure elle-meme (position du micro, moyennage), de la courbe cible, ou de la plage de frequence.

Comment regler sans se perdre

Vous debutez, ou vous voulez un resultat fiable : ne touchez a rien. Les defauts sont concus pour ca. Reglez seulement la plage de frequence si vous avez une raison precise de le faire.

Vous cherchez la meilleure qualite possible : cochez Beat-REW optimization (ce qui active du meme coup Reduce & repair et Critical band refinement), gardez Candidate placement, et essayez Refinement en comparant avant et apres. Attendez-vous a un calcul nettement plus long.

Vous voulez proteger votre materiel : gardez No boost below a 50 Hz, baissez Individual Max Boost, ou decochez Allow boosts pour travailler en attenuation seule.

Vous trouvez la correction artificielle a l'ecoute : c'est typiquement un exces de filtres etroits. Reglez Max Q in high band vers 2 ou 3, augmentez Notch exclusion pour que RCH ignore plus de creux, et verifiez que vous ne corrigez pas trop haut en frequence.

Vous voulez une correction plus simple : baissez Filters, augmentez Flatness target pour ne traiter que les gros defauts, et augmentez Min filter gain.

Gardez en tete le principe qui vaut pour tout RCH : c'est le cumul de la mesure corrigee et de votre chaine reelle qui doit suivre la cible. La sur-correction est l'erreur la plus frequente.

Caissons et MSO

Mesures de caisson faites dans REW

Si les mesures de caisson ont ete faites manuellement dans REW via le canal LFE, le filtre passe-bas du LFE de l'AVR peut biaiser la courbe. Dans ce cas seulement, utilisez Revert dans Subwoofer Tools pour annuler cet effet.

Si vos mesures viennent d'odd.wtf ou de l'application Audyssey, cette etape n'est en general pas necessaire.

Si vous mesurez via le chemin LFE de l'AVR, regler LPF for LFE a sa valeur la plus haute pendant la campagne de mesures simplifie generalement la compensation ensuite.

Flux de travail recommande pour les caissons

  1. Ajustez Additional Gain si vous voulez remonter ou descendre la courbe brute.
  2. Reglez Individual max boost et Overall max boost selon votre marge de securite.
  3. Si vous avez plusieurs caissons, utilisez Align Sub pour aligner les subs entre eux. Avec la case d'optimisation jointe, le meme passage calcule aussi des filtres par sub (voir plus bas).
  4. Utilisez Equalize Sub pour calculer les filtres d'egalisation.
  5. Utilisez Preview Sub pour afficher la reponse previsionnelle resultante.

Additional Gain ne change pas directement le niveau final envoye a l'AVR. Ce reglage deplace surtout la courbe de travail du sub par rapport a la cible pour faciliter l'optimisation. Individual max boost et Overall max boost definissent le gain positif maximum que l'algorithme peut utiliser.

Avec un seul sub, concentrez-vous surtout sur Equalize Sub puis Preview Sub. Align Sub devient utile seulement a partir de plusieurs caissons.

Optimisation jointe : filtres par sub

Avec plusieurs caissons, la case "Optimisation jointe : filtres par sub" change ce que calcule Align Sub. Au lieu d'aligner d'abord les subs puis de copier une meme EQ commune sur chacun, l'optimiseur joint cherche en meme temps les delais, les polarites, les niveaux par sub et jusqu'a trois filtres par sub, directement contre votre courbe cible — le meme principe qu'un projet MSO, mais integre a RCH.

Pourquoi c'est important : une EQ commune appliquee a tous les subs ne peut pas modifier la facon dont les subs interferent entre eux. Un filtre applique a un seul sub peut supprimer une contribution destructive a la source, ce qui suit generalement beaucoup mieux la cible dans la zone modale.

  • La recherche prend typiquement moins d'une minute sur un PC recent, quelques minutes sur un portable modeste ; un indicateur de progression a cote du bouton affiche la phase en cours et le pourcentage.
  • Individual max boost et Overall max boost sont respectes par les filtres par sub, exactement comme pour l'EQ commune.
  • Les ajustements de niveau par sub sont en attenuation seule : l'optimiseur reequilibre les subs, il ne les gonfle jamais au-dela de leur capacite reelle.
  • Chaque execution repart d'un etat propre : filtres, inversions, delais et ajustements appliques par une execution precedente sont d'abord remis a zero. Vos propres reglages manuels de niveau sont conserves. Vous pouvez donc relancer l'optimisation autant de fois que vous voulez.
  • Les filtres par sub sont ecrits dans les premiers emplacements de filtre de chaque sub en filtres manuels (non Auto), pour que les etapes suivantes ne les ecrasent jamais.

Dans ce mode, "Utiliser des filtres all-pass (plus lent)" autorise l'optimiseur a placer aussi un filtre all-pass par sub. Un all-pass tourne la phase localement sans toucher au niveau : c'est le seul levier capable de transformer une contribution destructive en contribution constructive au lieu de la couper. Il aide quand un creux d'annulation resiste aux delais, polarites et filtres ; l'optimiseur abandonne de lui-meme les all-pass qui ne rapportent rien. En cas de doute, lancez l'optimisation une fois avec et une fois sans la case, puis comparez les courbes predites.

Sans la case d'optimisation jointe, Align Sub garde son comportement historique : alignement seul, la case all-pass aidant alors la recherche d'alignement quand elle est cochee.

Courbes predite et theorique

RCH maintient un caisson virtuel par position d'ecoute. La mesure nommee "LFE predicted_P1" est la somme predite de tous les subs avec leurs reglages actuels, recalculee automatiquement apres chaque action sur les subs. Apres Align Sub, une seconde mesure nommee "LFE Max Sum Theo_P1" montre le plafond theorique : la somme que vous obtiendriez si chaque sub arrivait parfaitement en phase a la position d'ecoute.

Comparer les deux courbes indique quelle part du maximum theorique l'alignement actuel capture reellement. Une courbe predite proche de la reference Theo signifie que le travail d'alignement est termine ; un ecart persistant et localise en frequence signale generalement un creux d'interference a attaquer avec l'optimisation jointe ou l'option all-pass.

Equalize Sub apres l'optimisation

Equalize Sub calcule l'EQ commune sur la somme predite et la distribue a chaque sub. Les filtres par sub trouves par l'optimisation jointe, ainsi que les all-pass, sont preserves : les filtres communs sont ranges dans les emplacements restes libres sur tous les subs. C'est l'etape de finition recommandee quand les filtres par sub ne suffisent pas a suivre la cible de pres.

L'etape peut etre relancee sans risque : chaque passage recalcule l'EQ depuis la base de l'optimiseur au lieu de s'empiler sur le resultat precedent, donc deux executions donnent le meme resultat.

Le reglage "Gain min de filtre" est applique partout : les filtres calcules sous le seuil sont ecartes pour que des corrections quasi nulles ne gaspillent pas d'emplacements de filtre.

MSO

Si vous travaillez avec plusieurs caissons et MSO, RCH sait exporter les mesures de sub au bon format pour initialiser un projet MSO, puis reimporter dans REW un export Equalizer APO provenant de MSO.

Ce bloc n'apparait que dans les configurations multi-sub et suppose que REW soit connecte et que les mesures portent les bons noms.

Principe simple :
1. Exporter les mesures de subs depuis RCH
2. Travailler dans MSO
3. Exporter depuis MSO au format Equalizer APO
4. Importer ce fichier texte dans RCH

Le menu d'export MSO a utiliser pour ce flux est illustre ci-dessous. Cliquez sur la miniature pour voir l'image en taille complete.

Si vous devez terminer l'import a la main dans REW, utilisez la fenetre EQ avec le fichier texte genere pour ce projet.

Cas particulier des mesures ADY avec plusieurs subs

Pour des mesures issues d'odd.wtf ou de l'application Audyssey avec plusieurs caissons, le mode directional bass est important. Sans lui, RCH peut ne pas disposer de mesures individuelles de sub suffisamment exploitables pour un vrai travail multi-sub.

Si l'import ADY multi-sub echoue ou affiche un avertissement sur ce point, il faut generalement refaire la session de mesures en mode Directional bass. Selon le modele d'AVR, RCH peut aussi vous renvoyer vers une methode manuelle REW ou vers un flux odd.wtf adapte a votre modele.

Finalisation

Une fois les previews valides, ouvrez la zone Finalization pour preparer l'export.

Si vous venez d'utiliser Find Sub Alignment, verifiez le champ LPF for LFE avant de creer le fichier OCA. RCH peut l'avoir releve automatiquement pour rester coherent avec la coupure de l'enceinte selectionnee.

Le bouton Find Best LFE Low-Pass, a cote du champ LPF for LFE, choisit cette valeur automatiquement. RCH teste toutes les frequences de passe-bas LFE disponibles pour votre AVR : pour chacune, il simule le filtre (LR24, le meme type que celui du bass management) sur la somme reelle des subs, prend en compte le decalage temporel que ce filtre impose au signal LFE, puis mesure la qualite de l'addition avec le canal complet des enceintes frontales (FL, C, FR) — la partie jouee par l'enceinte au-dessus de son crossover ET son grave redirige vers les subs — sur toute la bande utile du LFE, jusqu'a 120 Hz. La frequence retenue est celle qui donne la somme la plus constructive ; une valeur basse qui couperait du contenu sous 120 Hz est penalisee de la meme maniere qu'une annulation. Le detail par candidat (perte de sommation moyenne, pire creux, decalage pris en compte) est visible dans les logs, ainsi que la comparaison avec la valeur precedente. Les filtres des enceintes et la somme des subs doivent etre prets (memes prerequis que Find Best Crossover) ; aucun autre reglage n'est modifie.

La generation du fichier OCA est desormais entierement interne : RCH lit les filtres presents dans REW (y compris ceux que vous auriez regeneres ou ajustes directement dans son interface) et calcule lui-meme la reponse impulsionnelle au format de votre AVR. L'export est plus rapide et ne cree plus de mesures temporaires dans REW. Les types de filtres pris en charge sont les PK, les all-pass, les passe-bas/passe-haut (LP, HP, LP1, HP1), les notch, les shelves (LS, HS, LS 6dB, LS 12dB, HS 6dB, HS 12dB) et les filtres modaux — chacun reproduit exactement la sortie de REW. Si un autre type actif est present sur un canal (par exemple un Low pass/High pass de type crossover sur les emplacements 21/22), l'export s'arrete avec un message indiquant l'emplacement concerne : desactivez ou remplacez ce filtre avant de relancer.

L'export integre d'office aux filtres OCA de chaque enceinte avec bass management un passe-haut electrique Butterworth 12 dB/oct a la frequence de son crossover. Cascade avec le BW12 applique par l'ampli, le raccord avec le sub devient un couple LR24|LR24 symetrique et en phase — la bosse d'environ 2 dB au raccord disparait (voir l'etape 9). Les subs et les enceintes Large ne sont pas modifies, et le niveau exporte de chaque canal est preserve. Les canaux concernes sont listes dans les logs a chaque export. Note pour MultEQ (non XT/XT32) : avec un crossover sous 60 Hz, la reponse du filtre est legerement tronquee par la longueur limitee des filtres de l'ampli (un avertissement apparait dans les logs).

  • Choisissez le LPF for LFE.
  • Choisissez le mode de sortie du subwoofer.
  • Activez ou non les options dynamiques selon votre usage.
  • Choisissez le format OCA adapte a votre materiel.
  • Cliquez sur Create OCA File pour generer le fichier final.
  • Cliquez sur Create Settings File si vous voulez conserver une trace complete des reglages utilises.

Le fichier OCA est la sortie finale de RCH. Vous pourrez ensuite le charger dans votre AVR avec odd.wtf ou un outil compatible avec votre flux de travail.

Pour un flux standard, Subwoofer Output en mode LFE est la base recommandee. N'utilisez LFE + Main que si vous savez exactement pourquoi vous le voulez.

Le format OCA odd.wtf correspond au flux classique. Le format A1 Evo Acoustica est a reserver aux materiels ou workflows qui l'exigent, notamment quand odd.wtf n'est pas adapte au DSP cible.

Depannage

RCH ne se connecte pas a REW

  • Verifiez que REW est lance.
  • Verifiez que le serveur API est demarre.
  • Ne changez pas le port si vous n'avez pas une bonne raison de le faire.

Mes mesures n'apparaissent pas dans le tableau

  • En mode avance, verifiez le nommage des mesures dans REW.
  • Les noms doivent commencer par les noms Audyssey attendus, tels qu'ils sont exposes dans les details AVR.
  • Utilisez Reset Measurements si REW est dans un etat incoherent.

Align SPL donne un resultat incoherent

  • Chargez d'abord une courbe cible dans REW.
  • Assurez-vous d'etre sur la bonne position, en general la position moyennee.
  • Faites Time Align avant Align SPL.
  • Chargez la cible sous REW / EQ / Target settings / House curve.
  • Verifiez que le niveau de reference de la cible reste a 75 dB.

Le raccord enceinte-sub ne marche pas

  • Essayez une autre enceinte de reference dans Tuning.
  • Essayez une coupure plus haute.
  • Regenez le filtre de l'enceinte avant de relancer Find Sub Alignment.
  • Verifiez les previews avant d'exporter.

Le mode avance ne remplit pas correctement le tableau

  • Lancez REW vide, connectez RCH, puis ouvrez le MDAT apres Connect.
  • Verifiez que les noms des mesures commencent bien par les noms Audyssey attendus.
  • Utilisez Reset Measurements si REW a conserve un etat de travail precedent.

Le multisub ADY ne se laisse pas exploiter

  • Refaites la session de mesures en mode Directional bass.
  • Si RCH vous le demande, basculez vers la methode manuelle REW ou vers le flux odd.wtf adapte a votre AVR.

Une action est grisee ou echoue immediatement

  • Verifiez que REW est connecte avant toute operation de calcul.
  • Sans connexion API active, RCH peut charger des fichiers mais ne peut pas lancer les traitements.

odd.wtf ne trouve pas l'AVR

  • Verifiez que l'AVR est allume et sur le meme reseau que la machine qui lance odd.wtf.
  • Recuperez l'adresse IP via l'application Denon ou Marantz, puis relancez la commande avec l'option -i.
  • Sur macOS ou Linux, verifiez aussi que le binaire est executable avant de conclure a un probleme de detection reseau.

Vous voulez repartir de zero

Si vous voulez remettre toutes les mesures a leurs parametres par defaut, utilisez Reset Measurements. Si tout le projet doit etre vide, utilisez Reset Application.
