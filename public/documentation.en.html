Introduction

Room Correction Helper (RCH) is a calibration tool for Denon and Marantz home theater systems based on Audyssey. It works with REW to measure, align speakers and subwoofers, generate filters and verify previews, then transfers the calibration directly into your AVR through the RCH Bridge, a small local program that drives the receiver over the network (Audyssey measurement included).

RCH is not a one-click magic button. The goal of the application is to provide a guided but transparent process so you can verify each step before the final transfer.

This documentation describes the current behavior of the application and follows the commands visible in the interface.

This help page is integrated directly into the application. The instructions below assume that you are already inside RCH and can switch back to the Application tab at any time.

Before you start

Before using RCH, make sure the following points are covered:

  • You have a Denon or Marantz AVR compatible with Audyssey, powered on and connected to the same network as your computer.
  • You have installed a recent REW beta version.
  • The REW API server is enabled on port 4735.
  • You have downloaded and launched the RCH Bridge, the small local program that links RCH to the AVR. The per-platform binaries and first-launch instructions are in the Resources tab, RCH Bridge section. Just leave it running while you use RCH.

The REW / Bridge / AVR operational chain

RCH needs three links to work, shown at the top of the Application page as three indicators:

  1. REW: start REW with its API server, then click Connect in Main commands.
  2. Bridge: launch the RCH Bridge binary, then click Connect Bridge.
  3. AVR: once the bridge is connected, click Discover — when a single receiver is found on the network it is registered automatically (nothing else to do); when several receivers are found, pick one from the list then click Register AVR. You can also type the IP address manually and click Register AVR. The receiver model is identified automatically — it is never asked for. The Denon or Marantz mobile app is a quick way to retrieve the IP address if needed.

As long as the chain is incomplete (three green indicators), the application is deliberately locked: only the connection panels stay usable, and the banner lists the missing links. This is by design — the receiver configuration (model, MultEQ version, detected channels) is read live from the connected AVR, and every step of the workflow depends on it.

Browser note: recent Chrome and Edge ask for a "local network" permission the first time RCH connects to the bridge — accept it. Firefox works without any extra step. Safari is not supported with the https site (WebKit limitation): use Chrome, Edge or Firefox.

Practical advice: keep a clean working folder with your measurement files (MDAT, ADY, MQX, liveproject), your RCH session files and your target curves. This avoids many file-handling mistakes.

If you use a custom target curve, prepare it in advance as well. You will need it in REW for Align SPL and for final validation.

Choose your starting point

Whichever starting point you pick, the receiver configuration (model, MultEQ version, amp assignment, detected channels) always comes from the AVR connected through the bridge — never from a file. Files only provide measurements.

1. Measure with the built-in assistant (recommended)

The bridge drives the receiver to play the Audyssey sweeps on every speaker, and RCH imports each impulse response into REW as the measurement progresses. This is the most direct path: no file to handle, and the measurements arrive already named by channel and position. The full walkthrough is in Measuring with the assistant.

2. Import a measurement file (.ady, .mqx or .liveproject)

If you already measured with the Audyssey MultEQ Editor app (.ady), with MultEQ-X (.mqx) or with Dirac Live (.liveproject), drag and drop the file onto the drop zone of the Application page. The measurements are loaded automatically into REW.

These files provide measurements only: the configuration stays the one of the connected AVR. If the file does not match that receiver (different model, different MultEQ version), RCH shows a warning but lets the import proceed — it is up to you to judge whether the measurements are still relevant for the receiver in place.

3. Advanced mode: measure yourself in REW

If you measure manually in REW with your own mic, your measurement names must follow the Audyssey names of the detected channels of the connected AVR (visible in the AVR Configuration panel), otherwise they will not be matched to the correct channels in the measurement table.

The most reliable sequence is: start REW empty, enable its API, connect the full chain, and only then open the MDAT file in REW. Before calculating, also make sure your measurements are not carrying leftover EQ, delay, or inversion state.

Recommended preparation for manual REW measurements

Before the measurement session, return the AVR to a simple and predictable state:

  • Neutralize any previous correction: clear the Audyssey calibration in the AVR menu, or import flat correction curves from an MQX file saved flat.
  • Set the volume to 62 (absolute scale) or -18 dB: this gives roughly 75 dB on Front Right.
  • (Optional, depending on your listening preference) If you plan to use Dynamic EQ: switch it on with a 5 dB strength, all other Audyssey processing off.
  • Disable loudness management and turn off ECO mode.
  • For subwoofer measurements, either set LPF for LFE to 250 Hz and cancel the filter effect later with "Revert AVR LPF" in RCH, or assign the subwoofer XLR output to the center channel to bypass the LFE low-pass filter.
  • Check that all distances are set to 3 m (do not use the "set default" button) and all levels to 0 dB.

On the REW side:

  • Select the Java EXCL driver if your interface supports it.
  • 48 kHz sampling, 0-24000 Hz measurement range.
  • Measurement level at -12 dBFS — important so the Atmos channels play at the correct volume.
  • For an Atmos system, play the lossless TrueHD files (one per channel) with VLC. All the files are available from the Resources tab, Audio Files section: the WAV measurement sweep and the per-channel .mlp files, including the SWx and SWx +10 dB variants for subwoofers.
  • Acoustic timing reference on the Front Left channel (the same one the Atmos files use).

Measure every channel with the played sweeps, preferably several times and at several positions. Measure the main listening position last and leave the microphone in place: post-calibration checks will be easier.

4. Dirac Live import (.liveproject file)

RCH can also import Dirac Live projects exported as .liveproject. Load the file just like an ADY (drag and drop, or click the drop zone on the Application page). As with ADY, the automatic import into REW only happens when RCH is connected to REW.

Unlike an ADY, a .liveproject does not contain ready-to-use impulse responses. RCH reconstructs the true impulse responses (with phase and inter-channel delays) from the raw microphone recordings stored in the file, by deconvolution. The microphone calibration embedded in the project is already applied during reconstruction: do not load a microphone calibration file into REW for these measurements, or you will double-correct the treble.

The file may contain several measurement positions; RCH recovers all of them and names each measurement by channel and position. This processing is heavier than an ADY import, especially for a multi-position project: it runs in the background and can take from a few seconds to about a minute. Progress is shown in the log.

The relative levels and delays between channels are preserved (the closest speaker sits at time zero, the others offset by their real delay). The absolute distance is not recoverable, and subwoofers appear farther away because their delay includes the subwoofer DSP latency: this is normal.

Important: like the other files, a .liveproject only provides measurements. The configuration (channels, MultEQ version, options) is the one of the AVR connected through the bridge, and that is what drives the optimization and the final transfer.

Measuring with the assistant

The Audyssey Measurement section of the Application page drives a complete measurement session through the bridge: the receiver plays the Audyssey sweeps on every speaker, and RCH imports each impulse response into REW as the session progresses, already named by channel and position. The Audyssey microphone calibration is handled by the bridge: you have no calibration file to load.

The assistant is optional: you can just as well measure yourself in REW with your own microphone (see Choose your starting point, advanced mode) or import an existing measurement file.

Prepare the session

  • Plug the Audyssey microphone supplied with the receiver into its SETUP MIC jack.
  • Unplug any headphones and disconnect any Bluetooth transmitter.
  • Place the microphone at ear height at the main listening position, pointing at the ceiling, on a stand if possible.
  • Keep the room silent during each measurement.

Then click Start measurement session. If a precondition is not met (microphone not plugged, headphones plugged, receiver powered off...), RCH shows an explicit message: fix it and retry.

Measure the positions

Position 1 always measures the full channel plan detected by the receiver: it is the detection pass (speaker presence, polarity). Channel selection only applies from position 2 onward: you can then remeasure only a subset of channels if needed.

For each position: move the microphone, click Measure position, and wait for the sweeps to finish (progress is displayed). The responses are imported into REW as the measurement progresses. Any warnings (reversed polarity detected, asymmetry) are displayed but do not block the session: note them and check the physical wiring if an inversion is reported.

Set the subwoofer levels

For each detected subwoofer, the Subwoofer level matching block shows a live SPL reading: click Match level, turn the gain knob of the subwoofer until it reads about 75 dB, then click Stop. Preferably do this before measuring the positions, so the measurements capture the right level.

Complete or cancel

When all positions are measured, click Complete session to hand control back to the receiver. Cancel session abandons the session at any time.

Then save your measurements in REW (File / Save all measurements) as a .mdat file: that file is what keeps your measurements from one working session to the next — RCH stores no signal data (see Session).

Recommended workflow

Step 1. Connect the chain and load the measurements

  1. Start REW (API server enabled) and the RCH Bridge binary.
  2. Switch back to the Application tab.
  3. Complete the chain: Connect (REW), Connect Bridge, then register the AVR (see Before you start). The three indicators of the banner must be green.
  4. Load your measurements: run a session with the measurement assistant, or drag and drop an .ady, .mqx or .liveproject file, or measure yourself in REW / open your .mdat (advanced mode).

Once the AVR is registered, open the AVR Configuration panel and verify that the detected channels match your actual setup.

In advanced mode, avoid loading the MDAT before connecting. RCH populates its table more reliably when REW is already running, the API is active, and the file is opened once the chain is complete.

Step 2. Reset the calculation state when needed

Use Reset Measurements if you want to reset all measurements to their default parameters before running calculations again. Use Reset Application if you want to clear the whole project after a bad workflow, a wrong file, or a complete project change.

In practice, Reset Measurements clears smoothing, IR windows, target settings, room curve, generic equalizer, delays, and inversion state. It is the right first step before repeating a full workflow.

Step 3. Set speaker calculation options

In Speaker Tools, you can configure smoothing, windowing, boost limits, and frequency range. Smoothing and windowing drive the level of detail of the equalization — light smoothing produces detailed corrections with many filters, heavy smoothing subtle corrections with few filters — as well as how the curves are displayed. Subwoofer alignment computations (Align Sub, predicted sum, theoretical ceiling) always measure the raw response, regardless of these settings: your display choices cannot degrade alignment accuracy.

The Optimized MTW windowing isolates the direct sound and the room resonances: ideal for choosing what to correct. It does however lower the curve by 1 to 2 dB above 300 Hz; RCH now measures and compensates this offset automatically at calculation time, band by band — filter gains stay correctly dosed and the slope of your target curve is properly carried into the result (the applied offset is shown in the logs). If you prefer a curve that is detailed in the bass and smooth in the highs without any level change, variable smoothing is the recommended alternative.

In the same spirit, RCH corrects the bass finely and smooths the highs heavily: in the highs the direct sound is already well below the target, and not over-correcting it is the recommended practice. The frequency range you set is authoritative — RCH does not impose any zone. Keep in mind that it is the combined corrected measurement + real chain that must follow the target: avoid over-correction.

Step 4. Create averages

If you have measurements from several positions, start by averaging them with Averages. Three strategies are available: delete all original measurements, keep only position 1, or keep everything.

Keeping position 1 in addition to the averages is useful if you later want to compare the result at the main listening position without losing the averaged curves.

RCH offers three averaging methods: Vector average, RMS + phase avg., and dB + phase avg. If you do not already have a clear preference, pick one method and keep it consistent throughout the project so your comparisons stay meaningful.

Before each average, RCH now time-aligns the positions itself (envelope cross-correlation for the coarse estimate, then a constrained fine cross-correlation), replacing REW's Cross corr align which could slip by a full cycle on some measurements. The applied offsets are reported in the logs. This alignment improves the coherence between positions, especially for the Vector average and for subwoofers.

For filter calculation, prefer RMS + phase avg.: this method preserves the level at all frequencies. Vector average remains valuable for time alignment and for judging the coherence between positions, but it underestimates the measurement level (by -1 to -2 dB with 3 positions, up to -5 dB with 8 positions): filters computed on it will be under-dosed.

Step 5. Align speakers in time

Use Time Align to recalculate distances and bring the speakers into temporal alignment. This should be done before fine level adjustments and before working on speaker-to-subwoofer integration.

The time reference is no longer the impulse peak but the arrival time of the wavefront, estimated through the excess phase (REW's "Estimate IR delay" method): the peak can land on a reflection stronger than the direct sound and skew the distance by several meters, whereas the arrival stays stable. The resulting distances are therefore closer to the physical ones; when no impulse response is available, RCH falls back to the previous peak alignment (reported in the logs).

Once Time Align is done, the speakers no longer move: only the subwoofers will be shifted afterwards (Find Sub Alignment). Do not re-run speaker time alignment after this step without a reason, as the following steps depend on it.

Up-firing Atmos speakers (laid flat on the mains, aimed at the ceiling): the useful path is the ceiling bounce, not the small direct sound that leaks toward the microphone. Alignment and distance follow that bounce; check that the distance is consistent on these channels.

Step 6. Load the target curve, then run Align SPL

The target curve must be loaded in REW before using Align SPL. This step adjusts speaker levels relative to that target.

Important: run Align SPL on the correct position. In practice, this usually means the averaged position, not an isolated raw measurement.

If RCH cannot find your target curve, load it in REW under EQ / Target settings / House curve, then confirm that the reference level is still set to 75 dB.

Step 7. Set crossovers and generate filters

In the measurement table, choose a global crossover or a per-group crossover, then use the Create buttons on each line or All Filters to generate everything at once.

The goal is to choose realistic crossover values based on the speaker response and on the practical quality of the speaker-to-subwoofer handoff. A crossover set too low can make alignment harder and produce a poor blend.

For a symmetrical pair such as FL and FR, prefer using the same crossover unless the measurements clearly justify doing otherwise.

Filters are computed by RCH. If you prefer to generate or adjust them directly in REW (EQ / Match target, or manual editing), RCH detects that the filters changed and uses the ones present in REW for the rest of the process: previews, sub alignment and transfer.

The calculation itself is driven by the Settings panel. The defaults are designed to give a good result without touching anything: if you want to understand what happens when you click Create, or fine-tune the behaviour, everything is detailed in Filter calculation (AutoEQ).

Find the best crossover automatically

Next to the crossover list, the wand button runs an automatic search for the best crossover of the speaker group. RCH tests every candidate frequency, computes the required shift of each speaker in the group (the same measurement as the recompute button in the Required shift column), then automatically selects in the list the one that minimizes the average absolute required shift of the group speakers. An identical button in the column header runs the search for every group at once.

Because the crossover is a group property, the per-line button only appears on the first speaker of each group. Clicking it processes the whole group.

Requirements: filters must already be generated and the subwoofer sum must exist (Sum subs / LFE predicted), otherwise RCH shows an alert and does nothing. In practice: generate filters once with a starting crossover, build the subwoofer sum, then run the best-crossover search and regenerate the filters for the chosen value.

If the best crossover requires a polarity inversion, RCH applies it right away, and always consistently across all speakers of the group: a frequency that would invert only one speaker of a pair is discarded. The log details, for each tested frequency, the shift of each speaker, the average, any inversion (inv), and the discarded frequencies.

Changing the crossover means downstream steps must be redone (filters, Find Sub Alignment, previews): the log reminds you of this.

Step 8. Align the subwoofer with a speaker

In Tuning, select the reference speaker and use Find Sub Alignment. The selected speaker must already have a computed filter. In practice, most users start with FL, FR, or the center speaker.

If the result is poor, try another speaker or raise the crossover before launching the alignment search again.

Find Sub Alignment moves the whole subwoofer block by a single delay, within a bounded range (about half a crossover period), and deliberately advances the sub by one sixteenth of a period. A small non-zero required shift left after the alignment is therefore normal.

When the search succeeds, RCH also updates LPF for LFE to a value consistent with the selected speaker, with a 120 Hz minimum. Still verify that field before the final transfer.

Step 9. Check the previews

Use the Preview buttons line by line, or All previews, to verify predicted speaker-plus-subwoofer responses before transfer. This is the best way to detect a poor handoff, a useful polarity change, or a wrong crossover choice.

If needed, Auto Adjust Inversion can help find a better polarity state for the subwoofer blend.

Note: if a speaker genuinely needs a polarity inversion to blend correctly, it may reveal a wiring mistake (+ and - swapped). Check it physically rather than hiding it with DSP.

If the previews look good on the average but less convincing at the main listening position, repeat the check while also keeping position 1 when creating averages.

Note on the handoff: the AVR's bass management is asymmetric — it applies an LR24 low-pass to the subwoofer but only a BW12 high-pass to the speaker, betting that the speaker's natural roll-off completes that BW12 — yet the EQ has just flattened that roll-off, which would push the sum about 2 dB above the target around the crossover. RCH corrects this by design: a second BW12 is baked into the transferred filters of every bass-managed speaker, so the real junction becomes a symmetric LR24|LR24 pair, in phase at every frequency. Previews and Find Best LFE Low-Pass simulate that complete chain: what the preview shows is what will come out of the AVR. The alignment measurements (Check, Find Sub Alignment, Find Best Crossover) work on the raw equalized curves — the LR24|LR24 pair being in phase at every frequency, it does not invalidate those measurements. Point to watch in the previews: below the crossover, where the speaker rolls off naturally and the EQ could not compensate, the second BW12 slightly deepens the dips — that is the region where the subs dominate, and a well-chosen crossover (Find Best Crossover) avoids the unfavourable pairings.

Filter calculation (AutoEQ)

When you click Create, RCH computes the parametric filters for the measurement itself: this is the AutoEQ engine. This section explains what it does, what each setting changes, and how to read its verdict. You do not need to change anything to get a good result: the defaults are a proven compromise across the reference measurements. Read on when you want to understand, or to fine-tune.

What the engine does

The principle is simple: RCH compares your measurement against the target curve and places filters where the deviation justifies it. In detail, the calculation always runs in this order:

  1. Deviation analysis. RCH restricts the measurement to the frequency range you chose, maps the target onto the same grid, then identifies the zones that deviate from the target. Very deep dips are set aside: they are acoustic cancellations, and correcting them achieves nothing.
  2. Filter placement. RCH places filters one at a time. At each step it recomputes what is left to correct, picks the most useful zone, places a filter there and optimises its gain and Q. It stops as soon as no zone justifies another filter, even if slots remain free.
  3. Cleanup and re-optimisation. Filters that became useless are removed, and the remaining ones are re-optimised together. This loop repeats until there is nothing left to remove.
  4. Overshoot reduction. If the correction pushes the response too far above the target somewhere, RCH lowers the boost filter responsible for it.
  5. Advanced options. Depending on the boxes you tick, RCH can replay the placement testing several hypotheses, then polish the result.

One important point: at every step, RCH only keeps the new result if it is better than the previous one. An option you enable therefore cannot degrade the result RCH keeps; at worst it changes nothing, in exchange for calculation time.

Where the settings live

The engine settings are in the Settings panel, AutoEQ section. Three further settings, located in Speaker Tools, weigh just as much on the result: the two boost limits and the frequency range. Those are the ones to look at first.

Individual Max Boost and Overall Max Boost

The first limits what a single filter may amplify; the second limits what all filters combined may amplify at the same frequency. Amplifying costs amplifier power and driver excursion: these two limits exist to protect your hardware. The defaults, 6 dB and 3 dB, are already a good compromise. Both fields accept 0 to 6 dB.

Frequency Range

The range you set is authoritative: RCH corrects nothing outside it and enforces no forbidden zone of its own. This is your main control. The default range stops at 16 kHz on purpose: in the treble the direct sound is already below the target, and a single-point measurement no longer represents what you hear. Extending to 20 kHz is possible, but rarely desirable.

The sliders, in screen order

Filters (default 20)

Maximum number of filters the engine may use. It is a ceiling, not a goal: RCH only places what is justified, and removes those that contribute nothing. Allowing many filters lets it correct narrow or numerous defects, but yields a more complex correction; few filters give a simpler, more robust one.

Max cut (dB) (default 25)

Maximum attenuation for one filter. Attenuating costs the amplifier nothing, unlike amplifying: that is why this limit is far wider than the boost limits. Correction by attenuation is the safest kind.

Flatness target (default 0.3)

The threshold, in dB, above which a deviation from the target is considered significant. This setting decides what the engine treats as a defect worth correcting. The lower the value, the smaller the deviations RCH addresses, so the more filters it places; the higher, the more it focuses on large defects only. This threshold is also what eventually stops placement: when no zone exceeds it any more, placement ends.

Optimization passes (default 20)

Maximum number of passes of the cleanup + global re-optimisation loop (this is not a per-filter pass count). The loop exits by itself as soon as a pass has nothing left to remove, which almost always happens on the first one: raising this value generally changes neither the result nor the calculation time.

Gain sign lock (default 0.5)

The threshold, in dB, above which a filter's sign is locked: beyond it, a filter that cuts can no longer become a filter that boosts, and vice versa. This prevents a filter from oscillating between the two during optimisation. Low value = locks sooner, more stable behaviour; high value = more freedom left to the optimiser.

Notch exclusion (default 6)

The depth, in dB below the target, at which a dip is declared uncorrectable and excluded from the calculation. This is an important setting: most deep dips come from acoustic cancellation between the direct sound and a reflection, and filling them would demand enormous energy for no result, and possibly at your hardware's expense. High value = RCH tries to correct more dips; low value = it ignores more of them.

Min filter gain (default 0.4)

The minimum gain, in dB, for a filter to be kept and sent to REW. Below it, the filter is deemed decorative and dropped. This avoids cluttering the equaliser with filters that have no audible effect.

Refinement iterations (default 100)

Number of iterations for the final polishing pass. Only has an effect if the Refinement box is ticked.

No boost below (Hz) (default 50)

No filter placed below this frequency will be allowed to amplify: the zone becomes cut-only. This is a hardware protection. Below 50 Hz, a boost demands a lot of excursion from the driver and a lot of power from the amplifier, usually to correct a cancellation that will not be corrected anyway. Set 0 to disable the protection.

Overshoot penalty weight (default 0.3)

An overshoot here is an excursion above the target curve: the correction pushes the response above the target, which is often more audible than a dip. This slider discourages the optimiser from producing such excursions beyond 1 dB. The higher the weight, the more it avoids them, at the cost of accuracy elsewhere.

Overshoot reduction threshold (dB) (default 1.5)

The same concern, handled differently: once the filters are computed, any excursion above the target larger than this threshold is corrected by lowering the boost filter responsible. The previous slider prevents during the calculation, this one repairs afterwards. It is also the threshold beyond which the report raises a warning.

Max Q below 200 Hz and Max Q in high band (default 0 = off)

These two sliders cap the Q factor, that is, how narrow filters may be. A narrow filter is a resonator: the higher its Q, the longer it takes to decay, which can be heard as ringing. Beyond Q = 8 a corrective filter becomes audible in its own right: that is why the range stops there. Left at 0, these caps are inactive and only the internal rules apply.

The two bands do not pose the same problem. In the bass, a narrow filter that cuts a room mode is useful: it shortens the room's resonance rather than creating one. Capping too low below 200 Hz deprives you of that treatment. In the treble, however, a narrow filter mostly corrects a local interference that moves as soon as the microphone does: there, capping is generally a good idea. If you only set one, set the high-band cap, around 2 or 3.

High band starts at (Hz) (default 3000)

The frequency from which the Max Q in high band cap applies. The 3 kHz default matches the zone where single-point measurements stop being representative. Lower it if you want to extend that caution into the midrange.

The checkboxes

Beat-REW optimization (off by default)

Enables a more ambitious optimisation mode, aimed at final quality rather than strict parity with REW. Slower, potentially better.

Worth knowing: this box also governs the next two. Reduce & repair and Critical band refinement are indented under it and stay greyed out (with no effect) as long as Beat-REW optimization is off. If you want to benefit from them, this is the box to tick.

Reduce & repair (on, requires Beat-REW)

After a first solution, RCH tries to remove the least useful or riskiest filters, then re-optimise what remains. The goal is a cleaner correction at equal quality: fewer filters, same result.

Critical band refinement (on, requires Beat-REW)

Re-optimises three bands separately — modal bass, critical midrange and treble — instead of targeting a single global score. The idea is to balance quality better where the ear is most sensitive.

Candidate placement (on by default)

Instead of placing each filter on the first zone found, RCH tests several candidate zones and keeps the most promising one. In practice, RCH reruns the whole placement this way, and only replaces the first result if the second is better. This doubles the calculation time, but cannot degrade the result: hence it is on by default.

Modal seeding (LPC) (off by default)

Analyses the room modes below 400 Hz (the resonances that create bass bumps) and reruns the filter placement with centre frequencies pinned exactly on those modes, with an initial width measured on the actual bump. The result is only adopted if it beats the standard calculation: this option cannot degrade the result, it only adds computation time. On the reference examples it improves some channels (fewer overshoots, slightly better RMS) and has no effect on the others.

Refinement (off by default)

A final polishing pass on the full grid, whereas the rest of the calculation works on a lightened one. On the reference examples, this option improves RMS everywhere, but degrades overshoots in rooms with pronounced dips. It is therefore left off, to be enabled case by case, by comparing.

Vary Q above 200 Hz (off by default)

Makes the Q ceiling vary progressively with frequency above 200 Hz, instead of a fixed ceiling. This matches the corresponding REW preference and is left at the same default value as in REW.

Narrow filters below 200 Hz (on by default)

Allows narrow filters in the bass, where room resonances are narrow and pronounced. Unticking widens bass corrections, making them gentler but less effective on modes.

Allow boosts (on by default)

Allows RCH to create filters that amplify, to raise zones below the target. Unticking works by attenuation only, the safest approach.

Worth knowing: unticking this box greys out both boost limits (Individual Max Boost and Overall Max Boost) and the calculation behaves as if they were 0. Your values are not lost: they become active again as soon as you tick the box back.

Reset defaults

This button is not an acoustic setting: it resets every AutoEQ parameter to its default value. It is the right reflex if you got lost in the settings.

Reading the result

At the end of each calculation, RCH publishes a verdict in the logs, in the form [createPhaseMatchFilter] rapport: PASS, followed where applicable by the flagged filters. Three verdicts are possible:

  • PASS: nothing to report, the correction is sound.
  • WARN: the result is usable but deserves a look. Most often: a filter with a high Q (beyond 8 in the bass, 10 elsewhere), a boost above 3 kHz, a residual overshoot, an improvement below 10 %, or a combined boost above your global limit.
  • FAIL: not to be used as is. Either a filter exceeds a safety ceiling (Q above 10 in the bass, 12 elsewhere), or the correction degrades the response instead of improving it.

A FAIL is almost never a settings problem: it means the measurement is asking for something impossible. The right reflexes are to look at the measurement itself (microphone position, averaging), at the target curve, or at the frequency range.

How to set this up without getting lost

You are starting out, or you want a safe result: change nothing. The defaults are designed for that. Only set the frequency range if you have a specific reason to.

You are after the best possible quality: tick Beat-REW optimization (which also activates Reduce & repair and Critical band refinement), keep Candidate placement, and try Refinement while comparing before and after. Expect a noticeably longer calculation.

You want to protect your hardware: keep No boost below at 50 Hz, lower Individual Max Boost, or untick Allow boosts to work by attenuation only.

The correction sounds artificial to you: that is typically an excess of narrow filters. Set Max Q in high band around 2 or 3, raise Notch exclusion so RCH ignores more dips, and check that you are not correcting too high in frequency.

You want a simpler correction: lower Filters, raise Flatness target so only large defects are addressed, and raise Min filter gain.

Keep in mind the principle that applies to all of RCH: it is the sum of the corrected measurement and your actual chain that must follow the target. Over-correction is the most common mistake.

Subwoofers and MSO

Subwoofer measurements made in REW

If your subwoofer measurements were made manually in REW through the LFE channel, the AVR LFE low-pass filter can bias the curve. In that case only, use Revert in Subwoofer Tools to compensate for that effect.

If your measurements come from the measurement assistant or from the Audyssey app, this step is usually not needed.

If you measured through the AVR LFE path, using the highest LPF for LFE value during the measurement session generally makes the later compensation easier.

Recommended subwoofer workflow

  1. Adjust Additional Gain if you want to raise or lower the raw curve.
  2. Set Individual max boost and Overall max boost based on your safety margin.
  3. If you have multiple subs, use Align Sub to align them with each other. With the joint optimization checkbox, the same pass also computes per-sub filters (see below).
  4. Use Equalize Sub to compute EQ filters.
  5. Use Preview Sub to display the predicted resulting response.

Additional Gain does not directly change the final AVR trim. It mainly shifts the subwoofer working curve relative to the target so the optimizer can find a better solution. Individual max boost and Overall max boost define how much positive gain the algorithm is allowed to use.

With a single subwoofer, focus mainly on Equalize Sub and Preview Sub. Align Sub only becomes useful once you have multiple subwoofers.

Joint optimization: per-sub filters

With several subwoofers, the "Joint optimization: per-sub filters" checkbox changes what Align Sub computes. Instead of aligning the subs first and copying one shared EQ onto all of them afterwards, the joint optimizer searches delays, polarities, per-sub level trims and up to three per-sub filters at the same time, directly against your target curve — the same idea as an MSO project, but integrated into RCH.

Why it matters: a shared EQ applied to every sub cannot change the way the subs interfere with each other. A filter applied to one sub can remove a destructive contribution at its source, which usually tracks the target much better in the modal region.

  • The search typically takes under a minute on a recent PC, a few minutes on a modest laptop; a progress indicator next to the button shows the current phase and percentage.
  • Individual max boost and Overall max boost are honoured by the per-sub filters, exactly like they are for the shared EQ.
  • Per-sub level trims are attenuation only: the optimizer rebalances the subs, it never inflates them above their real capability.
  • Each run starts from a clean state: filters, inversions, delays and the trims applied by a previous run are reset first. Your own manual level adjustments are preserved. You can therefore re-run the optimization as many times as you want.
  • The per-sub filters are written into the first filter slots of each sub as manual (non Auto) filters, so later steps never overwrite them.

In this mode, "Use all pass filters (slower)" allows the optimizer to also place one all-pass filter per sub. An all-pass rotates the phase locally without touching the level: it is the only lever that can turn a destructive contribution into a constructive one instead of muting it. It helps when a cancellation dip resists the delay, polarity and filter levers; the optimizer automatically drops all-pass filters that do not pay for themselves. If you are unsure, run the optimization once with and once without the checkbox and compare the predicted curves.

Without the joint checkbox, Align Sub keeps its historical behavior: alignment only, with the all-pass checkbox helping the alignment search when enabled.

Predicted and theoretical curves

RCH maintains a virtual subwoofer per listening position. The measurement named "LFE predicted_P1" is the predicted sum of all subs with their current settings, recomputed automatically after each subwoofer action. After Align Sub, a second measurement named "LFE Max Sum Theo_P1" shows the theoretical ceiling: the sum you would get if every sub arrived perfectly in phase at the listening position.

Comparing both curves tells you how much of the theoretical maximum the current alignment actually captures. A predicted curve close to the Theo reference means the alignment job is done; a persistent gap localized in frequency usually points to an interference dip worth attacking with the joint optimization or the all-pass option.

Equalize Sub after the optimization

Equalize Sub computes the shared EQ on the predicted sum and distributes it to every sub. The per-sub filters found by the joint optimization, and the all-pass filters, are preserved: the shared filters are packed into the remaining free slots on every sub. This is the recommended finishing step when the per-sub filters alone are not enough to track the target closely.

The step is safe to repeat: each pass recomputes the EQ from the optimizer baseline instead of stacking on top of the previous result, so running it twice gives the same answer.

The "Min filter gain" setting is enforced everywhere: computed filters below the threshold are discarded so near-zero corrections do not waste filter slots.

MSO

If you work with several subwoofers and MSO, RCH can export sub measurements in a format suitable for initializing an MSO project, then reimport into REW an Equalizer APO export produced by MSO.

This block appears only for multi-sub configurations and assumes that REW is connected and that measurement names are correct.

Simple principle:
1. Export sub measurements from RCH
2. Work inside MSO
3. Export from MSO in Equalizer APO format
4. Import that text file back into RCH

The MSO export menu used for this workflow is shown below. Click the thumbnail to open the full-size image.

If you need to finish the import manually in REW, use the EQ window with the text file created for that project.

Special case: ADY measurements with multiple subs

For measurements imported from an ADY file with multiple subwoofers, directional bass mode matters: without it, the file may not contain enough usable individual subwoofer measurements for a real multi-sub workflow.

If a multi-sub ADY import fails or warns about this, repeat the measurement session with the built-in assistant (it measures each subwoofer individually), or repeat the Audyssey session in Directional bass mode, or measure each subwoofer manually in REW.

Directional bass mode is only needed for taking the measurements: after the calibration transfer, check that the subwoofer mode of the receiver is set to Standard — the listening mode the settings produced by RCH are built for.

If your measurements contain a single subwoofer channel (session done in Standard mode: the receiver then measures all subwoofers together, as a single sweep), RCH automatically copies that calibration (filter, gain, distance) to every detected subwoofer when saving the banks and building the transfer. This matches the official behaviour; individual subwoofer alignment (per-sub delays and gains) however requires measurements taken in Directional bass mode.

Finalization and transfer

Once the previews look correct, open the Finalization area to prepare the transfer of the calibration into the receiver. The transfer goes directly through the bridge: there is no file to generate and no external program to run.

At the top of the area, the Speaker Preset row shows the active speaker preset of the receiver (read live through the bridge) and lets you switch between Preset 1 and Preset 2: pick the one that will receive the calibration before transferring. On receivers without speaker presets, the row says so and the buttons are hidden.

If you have just used Find Sub Alignment, verify the LPF for LFE field before transferring. RCH may have raised it automatically to stay consistent with the selected speaker crossover.

The Find Best LFE Low-Pass button, next to the LPF for LFE field, picks this value automatically. RCH tests every LFE low-pass frequency available for your AVR: for each one, it simulates the filter (LR24, the same type as the bass management low-pass) on the real sum of the subwoofers, takes into account the time shift this filter imposes on the LFE signal, then measures how well the result adds up with the complete front channels (FL, C, FR) — the part played by the speaker above its crossover AND its bass redirected to the subwoofers — across the whole useful LFE band, up to 120 Hz. The selected frequency is the one giving the most constructive summation; a low value that would cut content below 120 Hz is penalised exactly like a cancellation. The per-candidate details (mean summation loss, worst dip, time shift taken into account) are visible in the logs, along with the comparison against the previous value. The speaker filters and the sub sum must be ready (same prerequisites as Find Best Crossover); no other setting is modified.

Calibration filter generation is fully internal: RCH reads the filters present in REW (including any you regenerated or adjusted directly in its interface) and computes the impulse response in your AVR's format by itself. Supported filter types are PK, all-pass, low/high pass (LP, HP, LP1, HP1), notch, the low/high shelves (LS, HS, LS 6dB, LS 12dB, HS 6dB, HS 12dB) and modal filters — each reproduces REW's output exactly. If another active type is present on a channel (for example a crossover-style Low pass/High pass on slots 21/22), the save stops with a message naming the slot: disable or replace that filter before retrying.

The transferred filters of every bass-managed speaker bake in an electrical Butterworth 12 dB/oct high-pass at the channel's crossover frequency. Cascaded with the BW12 the AVR applies, the junction with the subwoofer becomes a symmetric, in-phase LR24|LR24 pair — the ~2 dB bump at the handoff disappears (see step 9). Subwoofers and Large speakers are untouched, and each channel's level is preserved. The affected channels are listed in the logs. Note for MultEQ (non XT/XT32): with a crossover below 60 Hz the filter response is slightly truncated by the AVR's limited filter length (a warning is logged).

The two filter banks: Reference and Flat

An Audyssey receiver stores two filter banks, Reference and Flat, which you switch between while listening in the Audyssey menu of the receiver. RCH lets you fill each of the two with your own calculations. Key point of the Audyssey format: the receiver only carries one set of delays, levels, distances and crossovers, shared by both banks — only the filters differ from one bank to the other.

The recommended path:

  1. Run the full workflow (averages, alignments, subwoofer optimization, filter generation, previews) with your main target curve.
  2. Click Save current filters on the Reference bank: RCH captures the filters of every channel.
  3. Change the target curve, and nothing else (a flat target for example), then recompute the filters (All Filters, Equalize Sub).
  4. Click Save current filters on the Flat bank. Since delays, levels and crossovers are shared by both banks, RCH refuses to save the second bank if anything other than the filters changed since the first one — the message then lists the detected differences.

The Duplicate to the other bank button is a shortcut: it copies the already saved bank into the other one, to transfer the same filters on both sides without recomputing. Clear banks empties both banks to start over.

Transfer to the receiver

The transfer requires both banks to be saved. Before transferring:

  • Choose the LPF for LFE value.
  • Choose the subwoofer output mode: LFE is the recommended baseline; use LFE + Main only if you explicitly know why you need it.
  • Set the Audyssey option cards: MultEQ (on/off and Reference or Flat listening curve), Dynamic Volume (Low/Medium/High), Dynamic EQ (reference offset 0/5/10/15 dB) and Low Frequency Containment (amount 1-7). Each secondary setting only applies when its toggle is enabled.
  • Choose the final Subwoofer Mode (Standard or Directional): this is the mode the receiver ends up in after the transfer — Directional is only needed for taking measurements, Standard is the recommended listening mode.

First click Validate on AVR: the receiver checks the complete calibration without writing anything (dress rehearsal). Then Transfer to AVR starts the real transfer: progress is shown per channel, and Cancel transfer lets you interrupt — cancellation can take a few seconds if the receiver is finalizing. The amp assignment is read live from the receiver when the archive is built, so there is no possible mismatch between the transfer and the actual state of the receiver. If delays, levels or crossovers changed since a bank was saved, RCH flags that bank as stale: save it again before transferring.

Once the transfer completes, check the result by ear and in the receiver menu: MultEQ active, and switching between Reference and Flat available.

Two complementary buttons: Download .rch.json downloads the calibration archive that was built (useful for inspection or support), and Create Settings File generates a full text record of the settings used.

Session

The Session section of the Application page saves your work so you can resume it later. Export session downloads a .json file containing the settings, the working state of the measurements (without signal data) and the filter banks. Import session restores that file.

RCH also saves your state continuously in the browser (auto-save): a simple page reload loses nothing. The session file is for archiving a project, resuming it on another machine, or after a browser cleanup.

The measurements themselves live in REW: save them as a .mdat file (File / Save all measurements). To resume a project, reload the .mdat in REW first, then import the session: each measurement of the session is re-attached by identifier to the matching REW measurement. Session measurements that cannot be found in REW are reported (title and position) — reload the right .mdat and import again if that list is not empty.

The existing side exports (subwoofer measurements for MSO, CamillaDSP conversion) are kept and independent from the session file.

Troubleshooting

RCH does not connect to REW

  • Make sure REW is running.
  • Make sure the API server is started.
  • Do not change the port unless you have a good reason to do so.

RCH does not connect to the bridge

  • Make sure the RCH Bridge binary is launched and still running (terminal window open).
  • On Chrome or Edge, accept the "local network" permission asked on first connection. If you declined it, re-allow it in the site settings.
  • Safari is not supported with the https site: use Chrome, Edge or Firefox.
  • If the bridge seems stuck, the Reset bridge button (AVR / Bridge actions menu) puts it back into a clean state.

The bridge does not find the AVR

  • Make sure the receiver is powered on and on the same network as the machine running the bridge.
  • Try Discover for an automatic network scan; otherwise retrieve the IP address from the Denon or Marantz app, enter it, then Register AVR.
  • If the receiver is flagged busy, a measurement or a transfer is in progress: wait for it to finish, the connection stays valid.

The application is locked (grayed sections)

  • The REW / Bridge / AVR chain is incomplete: the banner at the top of the page lists the missing links.
  • Connect REW, connect the bridge, then register the AVR. The unlock is immediate as soon as the three indicators are green.

My measurements do not appear in the table

  • In advanced mode, verify the measurement names in REW.
  • The names must start with the expected Audyssey names shown in the AVR details.
  • Use Reset Measurements if REW is in an inconsistent state.

Align SPL gives an incoherent result

  • Load a target curve in REW first.
  • Make sure you are on the correct position, usually the averaged position.
  • Run Time Align before Align SPL.
  • Load the curve under REW / EQ / Target settings / House curve.
  • Confirm that the target reference level is still 75 dB.

The speaker-subwoofer handoff does not work well

  • Try another reference speaker in Tuning.
  • Try a higher crossover value.
  • Regenerate the speaker filter before retrying Find Sub Alignment.
  • Check the previews before transferring.

Advanced mode does not populate the table correctly

  • Start REW empty, connect RCH, then open the MDAT after Connect.
  • Make sure measurement names begin with the expected Audyssey names.
  • Use Reset Measurements if REW is carrying over a previous working state.

Multi-sub ADY cannot be used properly

  • Repeat the measurement session with the built-in assistant (each subwoofer is measured individually).
  • Or repeat the Audyssey session in Directional bass mode, or measure each subwoofer manually in REW.

The transfer fails or seems stuck

  • Run Validate on AVR first: the dry-run error message points at the problem without writing anything into the receiver.
  • If a bank is flagged stale, delays, levels or crossovers changed since it was saved: save both banks again on consistent states.
  • A cancellation during the final phase can take a few seconds: let the transfer finish cleanly.
  • If the receiver stays in a strange state after an incident, power it off and on again before retrying.

An action is disabled or fails immediately

  • Check the REW / Bridge / AVR chain: the three indicators must be green.
  • Without a complete chain, RCH only allows the connection panels.

You want to start over

If you want to reset all measurements to their default parameters, use Reset Measurements. If you want to clear the whole project, use Reset Application.
