Introduction

Room Correction Helper (RCH) is a calibration tool for Denon and Marantz home theater systems based on Audyssey. It works with REW to import measurements, align speakers and subwoofers, generate filters, verify previews, and produce an OCA file that can be used with your AVR.

RCH is not a one-click magic button. The goal of the application is to provide a guided but transparent process so you can verify each step before the final export.

This documentation describes the current behavior of the application and follows the commands visible in the interface.

This help page is integrated directly into the application. The instructions below assume that you are already inside RCH and can switch back to the Application tab at any time.

Before you start

Before using RCH, make sure the following points are covered:

  • You have a Denon or Marantz AVR compatible with Audyssey.
  • You have installed a recent REW beta version.
  • The REW API server is enabled on port 4735.
  • All calculation steps require an active REW connection. Without it, RCH can still inspect some files and produce exports, but it cannot run processing steps.
  • You have a supported entry point for the current workflow: ADY for the guided mode, or AVR + REW measurements for the advanced mode.
  • For advanced use, you already know how to measure correctly in REW with a measurement microphone or an equivalent method.

If you plan to generate and then load an OCA file into the AVR, odd.wtf remains the most natural companion tool for that workflow.

Platform-specific odd.wtf binaries, lossless sweeps, and Atmos files remain centralized in the Resources tab. This page therefore focuses on when to use them, not on duplicating the full download catalog.

Practical advice: keep a clean working folder with your ADY, MQX, AVR, MDAT, OCA, and target curve files. This avoids many protocol and file-handling mistakes.

If you use a custom target curve, prepare it in advance as well. You will need it in REW for Align SPL and for final validation.

Choose your starting point

1. Basic mode with an ADY file

This is the simplest entry point. You measure with odd.wtf or with the Audyssey MultEQ Editor app, then import the ADY file into RCH. When RCH is connected to REW, the measurements are loaded automatically into REW.

With odd.wtf, the simplest path is the built-in measurement workflow that directly produces an ADY file. If you prefer to measure manually in REW with your own mic, keep odd.wtf only for generating the AVR file used by advanced mode.

If REW is not connected, RCH can still extract the measurements from the file, but the automatic import into REW will not happen.

2. Special case: MQX files

RCH can also process MQX files, but this path is not currently the most robust standard workflow in the Application page.

In practice, you should mainly rely on ADY for the simple workflow and AVR + REW for the advanced workflow. If you work with MQX, treat it as an advanced case that should be validated carefully for your project and AVR context.

To use this path, load AVR data first. RCH needs it to rebuild the channel mapping correctly.

3. Advanced mode with REW

If you measure manually in REW, you must first generate an AVR file with odd.wtf, then load that AVR file into RCH. Your REW measurement names must follow the Audyssey names exposed by the AVR file, otherwise they will not be matched to the correct channels in the measurement table.

The most reliable sequence is: start REW empty, enable its API, connect RCH, and only then open the MDAT file in REW. Before calculating, also make sure your measurements are not carrying leftover EQ, delay, or inversion state.

Recommended preparation for manual REW measurements

Before the measurement session, return the AVR to a simple and predictable state. The minimum useful checklist is:

  • Remove or neutralize any previous correction before measuring.
  • Set the volume around 62 or -18 dB, with roughly 75 dB on Front Right.
  • Disable unnecessary processing during measurement, especially ECO mode and loudness processing.
  • If you measure subwoofers through the LFE path, temporarily raise LPF for LFE to its maximum value, or plan to use Revert later inside RCH.
  • For an Atmos system, use the matching lossless sweeps from Resources, with Front Left as the acoustic timing reference.

On the REW side, keep a stable setup: Java EXCL if your interface supports it, 48 kHz sampling, 0-24000 Hz sweep range, and -12 dBFS sweeps so Atmos channels stay level-consistent.

Generate an AVR file with odd.wtf

In advanced mode, the AVR file is the naming reference that lets RCH match REW measurements to the correct Audyssey channels.

  1. Download the binary that matches your OS from the Resources tab and place it in a stable folder.
  2. On macOS or Linux, make it executable if needed, then start with a help command to confirm that it launches correctly.
  3. Use the network search command to detect the AVR, then run the generation command to produce the AVR file that you will import into RCH.
  4. If automatic detection fails, rerun the generation command while specifying the AVR IP address directly.
odd search
odd gen
odd gen -i 192.168.x.x

The exact binary name depends on your platform. Adapt the command to odd.wtf, odd.mac, or odd.linux as needed. The Denon or Marantz mobile app is usually the fastest way to retrieve the AVR IP address.

Recommended workflow

Step 1. Connect REW and load the project

  1. Start REW and enable its API server.
  2. Switch back to the Application tab.
  3. Click Connect in Main Commands.
  4. Drag and drop your ADY or AVR file.
  5. If you are in advanced mode, then open your measurement set in REW so RCH can map the measurements correctly.

Once the project is loaded, open the AVR details and verify that the detected channels match your actual setup.

In advanced mode, avoid loading the MDAT before connecting. RCH populates its table more reliably when REW is already running, the API is active, and the file is opened only after clicking Connect.

Step 2. Reset the calculation state when needed

Use Reset Measurements if you want to reset all measurements to their default parameters before running calculations again. Use Reset Application if you want to clear the whole project after a bad workflow, a wrong file, or a complete project change.

In practice, Reset Measurements clears smoothing, IR windows, target settings, room curve, generic equalizer, delays, and inversion state. It is the right first step before repeating a full workflow.

Step 3. Set speaker calculation options

In Speaker Tools, you can configure smoothing, windowing, boost limits, and frequency range. These settings determine how RCH and REW will prepare the next calculations.

Step 4. Create averages

If you have measurements from several positions, start by averaging them with Averages. Three strategies are available: delete all original measurements, keep only position 1, or keep everything.

Keeping position 1 in addition to the averages is useful if you later want to compare the result at the main listening position without losing the averaged curves.

RCH offers three averaging methods: Vector average, RMS + phase avg., and dB + phase avg. If you do not already have a clear preference, pick one method and keep it consistent throughout the project so your comparisons stay meaningful.

Step 5. Align speakers in time

Use Time Align to recalculate distances and bring the speakers into temporal alignment. This should be done before fine level adjustments and before working on speaker-to-subwoofer integration.

Step 6. Load the target curve, then run Align SPL

The target curve must be loaded in REW before using Align SPL. This step adjusts speaker levels relative to that target.

Important: run Align SPL on the correct position. In practice, this usually means the averaged position, not an isolated raw measurement.

If RCH cannot find your target curve, load it in REW under EQ / Target settings / House curve, then confirm that the reference level is still set to 75 dB.

Step 7. Set crossovers and generate filters

In the measurement table, choose a global crossover or a per-group crossover, then use the Create buttons on each line or All Filters to generate everything at once.

The goal is to choose realistic crossover values based on the speaker response and on the practical quality of the speaker-to-subwoofer handoff. A crossover set too low can make alignment harder and produce a poor blend.

For a symmetrical pair such as FL and FR, prefer using the same crossover unless the measurements clearly justify doing otherwise.

Step 8. Align the subwoofer with a speaker

In Tuning, select the reference speaker and use Find Sub Alignment. The selected speaker must already have a computed filter. In practice, most users start with FL, FR, or the center speaker.

If the result is poor, try another speaker or raise the crossover before launching the alignment search again.

When the search succeeds, RCH also updates LPF for LFE to a value consistent with the selected speaker, with a 120 Hz minimum. Still verify that field before the final export.

Step 9. Check the previews

Use the Preview buttons line by line, or All previews, to verify predicted speaker-plus-subwoofer responses before export. This is the best way to detect a poor handoff, a useful polarity change, or a wrong crossover choice.

If needed, Auto Adjust Inversion can help find a better polarity state for the subwoofer blend.

If the previews look good on the average but less convincing at the main listening position, repeat the check while also keeping position 1 when creating averages.

Subwoofers and MSO

Subwoofer measurements made in REW

If your subwoofer measurements were made manually in REW through the LFE channel, the AVR LFE low-pass filter can bias the curve. In that case only, use Revert in Subwoofer Tools to compensate for that effect.

If your measurements come from odd.wtf or from the Audyssey app, this step is usually not needed.

If you measured through the AVR LFE path, using the highest LPF for LFE value during the measurement session generally makes the later compensation easier.

Recommended subwoofer workflow

  1. Adjust Additional Gain if you want to raise or lower the raw curve.
  2. Set Individual max boost and Overall max boost based on your safety margin.
  3. If you have multiple subs, use Align Sub to align them with each other.
  4. Use Equalize Sub to compute EQ filters.
  5. Use Preview Sub to display the predicted resulting response.

Additional Gain does not directly change the final AVR trim. It mainly shifts the subwoofer working curve relative to the target so the optimizer can find a better solution. Individual max boost and Overall max boost define how much positive gain the algorithm is allowed to use.

The Use all pass filters checkbox is intended for multi-sub mode only. It can help achieve better alignment, but it also makes the calculation heavier.

With a single subwoofer, focus mainly on Equalize Sub and Preview Sub. Align Sub only becomes useful once you have multiple subwoofers.

MSO

If you work with several subwoofers and MSO, RCH can export sub measurements in a format suitable for initializing an MSO project, then reimport into REW an Equalizer APO export produced by MSO.

This block appears only for multi-sub configurations and assumes that REW is connected and that measurement names are correct.

Simple principle:
1. Export sub measurements from RCH
2. Work inside MSO
3. Export from MSO in Equalizer APO format
4. Import that text file back into RCH

The MSO export menu used for this workflow is shown below. Click the thumbnail to open the full-size image.

If you need to finish the import manually in REW, use the EQ window with the text file created for that project.

Special case: ADY measurements with multiple subs

For measurements coming from odd.wtf or from the Audyssey app with multiple subwoofers, directional bass mode matters. Without it, RCH may not have enough usable individual subwoofer measurements for a real multi-sub workflow.

If a multi-sub ADY import fails or warns about this, you usually need to repeat the measurement session in Directional bass mode. Depending on the AVR model, RCH may also direct you toward a manual REW method or an odd.wtf workflow variant for that model.

Finalization

Once the previews look correct, open the Finalization area to prepare the export.

If you have just used Find Sub Alignment, verify the LPF for LFE field before creating the OCA file. RCH may have raised it automatically to stay consistent with the selected speaker crossover.

  • Choose the LPF for LFE value.
  • Choose the subwoofer output mode.
  • Enable or disable dynamic options depending on your use case.
  • Choose the OCA format that matches your hardware.
  • Click Create OCA File to generate the final file.
  • Click Create Settings File if you want to keep a full text record of the settings used for this export.

The OCA file is the final output of RCH. You can then load it into your AVR with odd.wtf or another tool that matches your workflow.

For a standard workflow, Subwoofer Output in LFE mode is the recommended baseline. Use LFE + Main only if you explicitly know why you need it.

The odd.wtf OCA format matches the standard upload flow. The A1 Evo Acoustica format is for hardware or workflows that require it, especially when odd.wtf is not suited to the target DSP.

Troubleshooting

RCH does not connect to REW

  • Make sure REW is running.
  • Make sure the API server is started.
  • Do not change the port unless you have a good reason to do so.

My measurements do not appear in the table

  • In advanced mode, verify the measurement names in REW.
  • The names must start with the expected Audyssey names shown in the AVR details.
  • Use Reset Measurements if REW is in an inconsistent state.

Align SPL gives an incoherent result

  • Load a target curve in REW first.
  • Make sure you are on the correct position, usually the averaged position.
  • Run Time Align before Align SPL.
  • Load the curve under REW / EQ / Target settings / House curve.
  • Confirm that the target reference level is still 75 dB.

The speaker-subwoofer handoff does not work well

  • Try another reference speaker in Tuning.
  • Try a higher crossover value.
  • Regenerate the speaker filter before retrying Find Sub Alignment.
  • Check the previews before exporting.

Advanced mode does not populate the table correctly

  • Start REW empty, connect RCH, then open the MDAT after Connect.
  • Make sure measurement names begin with the expected Audyssey names.
  • Use Reset Measurements if REW is carrying over a previous working state.

Multi-sub ADY cannot be used properly

  • Repeat the measurement session in Directional bass mode.
  • If RCH asks for it, switch to the manual REW method or to the odd.wtf workflow variant that matches your AVR.

An action is disabled or fails immediately

  • Confirm that REW is connected before any calculation step.
  • Without an active API connection, RCH can load files but cannot run processing.

odd.wtf does not find the AVR

  • Make sure the AVR is powered on and on the same network as the machine running odd.wtf.
  • Retrieve the IP address from the Denon or Marantz app, then rerun the command with the -i option.
  • On macOS or Linux, also verify that the binary is executable before assuming this is only a network discovery problem.

You want to start over

If you want to reset all measurements to their default parameters, use Reset Measurements. If you want to clear the whole project, use Reset Application.
